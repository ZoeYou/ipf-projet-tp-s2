Programmation fonctionnelle : TP 4

Exercice 1 [Arbres binaires]

Chaque nœud d'un arbre binaire polymorphe possède une valeur, un sous-arbre gauche et un sous-arbre droit.

  1. Définir bintree le type des arbres binaires polymorphes
  2. Écrire la fonction map_tree: ('a -> 'b) -> 'a bintree -> 'b bintree qui renvoie un arbre dans lequel toutes les valeurs ont été remplacées par leur image par la fonction donnée en premier argument.
  3. Écrire la fonction fold_tree: ('a -> 'b -> 'b) -> 'a bintree -> 'b -> 'b qui est la fonction équivalente à List.fold_right pour les arbres binaires. L'exploration de l'arbre est faite dans l'ordre suivant : sous-arbre droit, racine puis sous-arbre gauche.
  4. Écrire une fonction is_abr: int bintree -> bool vérifiant qu'un arbre binaire est un abr.

Exercice 2

On définit les expressions arithmétiques avec des liaisons locales : 
type expr =
      | Nb of int | Plus of expr * expr
      | Mult of expr * expr | Minus of expr * expr
  1. Représenter l’expression (1 + 4) ∗ 3 − 5 en utilisant le type précédent.
  2. Proposer une fonction nb_op : expr -> int qui calcule le nombre d'opérations arithmétiques a effectuer pour évaluer une expression.
  3. Écrire la fonction eval: expr → int qui calcule la valeur entière d’une expression.
  4. Proposer une fonction nb_null : expr -> int qui calcule le nombre de sous-expressions qui s'évaluent à 0.
